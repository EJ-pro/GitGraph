### 1. SQLAlchemy 데이터베이스 커넥션 풀 확장
* **파일**: [database.py](file:///c:/Users/Playdata/dev/Project/ChatFolio/backend/database/database.py)
* **내용**:
  50명의 요청이 몰릴 때, 풀 크기가 부족하여 발생하는 `QueuePool limit of size 5 overflow 10 reached, connection timed out` 에러를 예방하기 위해 커넥션 풀 설정을 오버플로우 한도까지 넉넉하게 확장했습니다.
```python
engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    pool_size=20,          # 기본 유지 커넥션 수
    max_overflow=30,       # 부하 발생 시 최대 초과 생성 커넥션 (합계 최대 50개 커넥션 지원)
    pool_timeout=60,       # 풀이 찼을 때 신규 연결 대기 최대 시간 (초)
    pool_recycle=1800      # 30분 주기로 DB 커넥션 재생성 (커넥션 고사 방지)
)
```

### 2. Redis 연결 서킷 브레이커 (Fail-Open) 구현
* **파일**: [redis_client.py](file:///c:/Users/Playdata/dev/Project/ChatFolio/backend/core/cache/redis_client.py)
* **내용**:
  캐싱 모듈에서 Redis 연결 실패 시 매 API 요청마다 `redis.from_url`을 호출하며 2초의 커넥션 타임아웃을 소모하는 문제를 해결했습니다. 30초 쿨다운을 가진 서킷 브레이커 회로를 구축하여 Redis 장애 시에도 사용자는 딜레이 없이 서비스를 계속 이용(Fail-Open)할 수 있습니다.
```python
_redis_disabled = False
_last_connect_time = 0.0
_RECONNECT_COOLDOWN = 30.0  # 재연결 시도 쿨다운

def get_redis() -> redis.Redis | None:
    global _client, _redis_disabled, _last_connect_time
    if _client is not None:
        return _client

    now = time.monotonic()
    if _redis_disabled and (now - _last_connect_time < _RECONNECT_COOLDOWN):
        return None  # Redis 연결 장애 상태이므로 즉시 None 반환 (Fail-Open)

    _last_connect_time = now
    try:
        _client = redis.from_url(
            REDIS_URL, 
            decode_responses=True, 
            socket_connect_timeout=2,
            max_connections=100  # 커넥션 고갈 방지
        )
        _client.ping()
        _redis_disabled = False
        return _client
    except Exception:
        _client = None
        _redis_disabled = True
        return None
```

### 3. Chroma Vector DB에 대한 프로젝트 단위 스레드 락 적용
* **파일**: [engine.py](file:///c:/Users/Playdata/dev/Project/ChatFolio/backend/core/rag/engine.py)
* **내용**:
  Chroma DB는 SQLite를 파일 스토리지 백엔드로 사용하므로 동시 쓰기/읽기 작업이 발생하면 `OperationalError: database is locked`가 발생합니다. 프로젝트 ID별로 독립된 스레드 Lock을 보장하여 데이터베이스 디렉터리 생성 및 텍스트 벡터화 저장 단계를 안전하게 보호했습니다.
```python
_chroma_locks = {}
_chroma_lock_mutex = threading.Lock()

def _get_project_lock(project_id) -> threading.Lock:
    if not project_id:
        return threading.Lock()
    with _chroma_lock_mutex:
        if project_id not in _chroma_locks:
            _chroma_locks[project_id] = threading.Lock()
        return _chroma_locks[project_id]

# _prepare_vector_db 메서드 내부 전체를 project_lock 블록으로 감싸서 보호
```
